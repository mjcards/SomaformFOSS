# Contributing to SomaForm

First — thank you. SomaForm exists to make serious swimming analysis available to every coach and swimmer, not just those with expensive proprietary systems. Every contribution moves that forward.

---

## Who We're Looking For

SomaForm needs people from different backgrounds:

- **Swimming coaches** — technique knowledge, real-world testing, feedback on what's useful poolside
- **Swimmers and athletes** — lived experience of what feedback actually helps
- **Sport scientists / biomechanists** — help us get the analysis right
- **Developers** — improve the app, add features, optimise performance
- **Translators** — make SomaForm work for non-English speaking coaches and swimmers

---

## Ways to Contribute

### Report a Bug
Open a GitHub Issue. Include: what you expected, what happened, your browser/device, and if possible a screenshot or video clip.

### Suggest a Feature
Open a GitHub Issue tagged `enhancement`. Describe the coaching problem you're trying to solve, not just the feature itself.

### Improve the Framework
The biomechanical framework in `docs/FRAMEWORK.md` should evolve with coaching knowledge. If you spot an inaccuracy, an omission, or a stroke-specific nuance we've missed — open a PR or issue.

### Contribute Code
1. Fork the repo
2. Create a branch: `git checkout -b feature/your-feature-name`
3. Make your changes
4. Test in a real browser (Chrome recommended)
5. Open a Pull Request with a clear description

### Test at the Pool
The most valuable thing you can do is use SomaForm with real swimmers and report back what works and what doesn't. Open an issue tagged `field-report`.

---

## Code Style

- Vanilla HTML/CSS/JS where possible — no build step, no framework required for the core app
- Comments in plain English, written for coaches as much as developers
- Accessibility matters — the poolside display must be readable in bright outdoor conditions

---

## Code of Conduct

This project is inclusive of coaches, athletes, and developers at all levels. Be respectful. Critique ideas, not people. If something seems wrong technically — say so clearly and constructively.
